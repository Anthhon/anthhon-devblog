Ponteiros em C: o que ninguém te conta

15 January 2025 ~5 min read Tags: C, ponteiros, memória

O que é um ponteiro, de verdade?

Todo mundo aprende a definição: "um ponteiro é uma variável que guarda um endereço de memória." Correto. Mas o que isso significa na prática?

Quando você escreve:

int x = 42;
int *ptr = &x;

ptr não é mágico. É um número inteiro de 8 bytes que contém um endereço — a localização de x na RAM.

Vendo na prática com GDB

Compile com símbolos de debug e inspecione diretamente na memória:

gcc -g -o exemplo exemplo.c
gdb ./exemplo
(gdb) break main
(gdb) run
(gdb) x/xw &x
0x7fffffffe59c: 0x0000002a

O valor 0x0000002a é 42 em hexadecimal. Você está lendo a RAM diretamente.

Aritmética de ponteiros

Quando você faz p + 1, o compilador avança o endereço em sizeof(*p) bytes — no caso de int, 4 bytes. Não 1 byte.

int arr[] = {10, 20, 30, 40};
int *p = arr;
printf("%d\n", *(p + 2)); // 30

Sobre o Livro

Descompilando nasceu da vontade de criar um recurso em português que realmente explique o que acontece por baixo do capô — desde a sintaxe C até o código de máquina que o processador executa.

O que você vai aprender

  • Fundamentos de C — ponteiros, memória, structs, unions e o modelo de memória real
  • Assembly x86/x86-64 — leitura e escrita de código de montagem, convenções de chamada
  • Compilação e linkagem — o que GCC faz com seu código, ELF binários, seções
  • Debugging profundo — GDB, análise de stack frames, detecção de memory leaks
  • Engenharia reversa básica — Ghidra, radare2, identificando padrões em binários
